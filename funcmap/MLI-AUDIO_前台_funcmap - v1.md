# MLI-AUDIO｜業務員錄音前台 Functional Map

**專案：** MLI 高齡錄音系統輔助宣讀專案
**系統名稱：** 業務員錄音前台
**使用者：** 保險業務員
**入口：** 網頁版（桌機瀏覽器）＋ 嵌入行銷系統 App（iPad）
**使用情境：** 業務員銷售保險商品需要錄音後上傳
**產出日期：** 2026-05-22

> **架構說明：** 
> - 前台為獨立網頁應用，與內勤後台共用同一後端服務、同一資料庫。
> - 系統串接（建議書系統、行動投保系統、BPM、行銷系統）均由後端統一處理。
> - 前台身份驗證依賴行銷系統 App 的 SSO / Token 機制，不另行實作獨立登入頁
>
> **重疊項目標示：** 標記 ※ 的功能同時出現於後台文件，兩端皆需實作但視角不同。

## 模組：P-1 錄音作業介面

業務員 iPad / 網頁端的完整錄音操作流程，從取號、分題引導、錄音控制，到完成上傳。

| 子功能代號 | 子功能名稱   | 說明                                                                             | 優先級       |
| ----- | ------- | ------------------------------------------------------------------------------ | --------- |
| F-101 | 建立錄音案件  | 提供業務員介面輸入案件資訊後取號                                                               | Must-have |
| F-102 | 題目文稿內容  | 題目文稿依案件資訊透過介接取得，若無介接資訊的通路或商品，可由業務員自行上傳題目文稿。畫面依題號逐題引導，題卡清楚區分未錄音 / 已錄音 / 已跳過三種狀態 | Must-have |
| F-103 | 業務員自行錄音 | 業務員唸題題型的「開始錄音」按鈕組，含狀態切換（錄音中 → 重新錄音）與已錄題卡底色反灰                                   | Must-have |
| F-104 | 自動播稿錄音  | 自動唸題題型的「播稿並錄音」按鈕組，點選後同時播放 TTS 音檔與啟動錄音（播放依賴 F-201 音檔預生成完成；需處理音檔尚未就緒時的等待提示）      | Must-have |
| F-105 | 播放/重錄確認 | 已錄音題卡重錄前跳出確認提示；另提供「播放」按鈕回聽已錄音檔（未錄音時按鈕反灰不可按）                                    | Must-have |
| F-106 | 跳過與還原控制 | 可跳過題型提供「此題跳過」按鈕，題卡底色及錄音按鈕反灰；誤點跳過後可還原為錄音（需確認提示）；已錄音題卡不可再選跳過                     | Must-have |
| F-107 | 即時儲存    | 每題錄製完成後立即上傳分段音檔至後端，重錄時新檔直接覆蓋舊檔，防範系統閃退或網路中斷造成音檔遺失                               | Must-have |
| F-108 | 送出檢核與合併 | 點選完成送出時先檢核所有題卡是否皆已完成（未完成則提示並阻止送出）；確認後後端將分題音檔合併為一個整合音檔，前台顯示送審完成狀態               | Must-have |

## 模組：P-2 語音轉譯引擎

取用 TTS 音檔與前台播放，支援業務員在錄音過程中自動播放題目供客戶聆聽。
語言與聲音選項、語速調整均依賴第三方 TTS 廠商。

| 子功能代號   | 子功能名稱     | 說明                                                               | 優先級          |
| ------- | --------- | ---------------------------------------------------------------- | ------------ |
| F-201 ※ | TTS 文字轉音檔 | 後端將文稿依題號預先生成 TTS 音檔後，前台按題次取用並播放；播放失敗時需有提示機制。※ 後台負責生成與管理，前台負責播放體驗 | Must-have    |
| F-202   | 多語言 TTS   | 支援國語 / 台語 × 男聲 / 女聲共 4 種聲音組合，前台可依案件設定選擇對應聲音播放（依賴第三方 TTS 廠商）      | Nice-to-have |
| F-203   | 語速調整      | 播放速度正常 1 倍，至少可調慢至 0.5 倍，照顧高齡客戶聆聽需求（依賴第三方 TTS 廠商）                 | Nice-to-have |

## 模組：P-3 案件查詢介面

業務員在錄音前台的日常案件查詢與操作。

| 子功能代號 | 子功能名稱  | 說明                                             | 優先級       |
| ----- | ------ | ---------------------------------------------- | --------- |
| F-301 | 錄音案件清單 | 提供業務員介面進行案件查詢，可即時查看案件資訊與審核狀態                   | Must-have |
| F-302 | 錄音案件內容 | 提供業務員介面進行案件內容檢視                                | Must-have |
| F-303 | 播放音檔   | 提供使用者聽取已錄製的音檔                                  | Must-have |
| F-304 | 案件搜尋   | 支援簡易快速查詢                                       | Must-have |
| F-305 | 修改案件資訊 | 提供業務員修改案件特定資訊                                  | Must-have |
| F-306 | 刪除音檔   | 提供業務員對退回或是尚未通過審核的音檔刪除                          | Must-have |
| F-307 | 上傳音檔   | 提供業務員對退回或是尚未通過審核重新上傳音檔，或在網頁版（桌機瀏覽器）建立的案件自行上傳音檔 | Must-have |

## 模組：P-4 外部系統整合服務

前台錄音流程所需的取號機制，以及與建議書系統、行動投保系統的資料交換。
實際 API 串接由後端執行，前台透過後端取得所需資料。

| 子功能代號   | 子功能名稱          | 說明                                                                    | 優先級       |
| ------- | -------------- | --------------------------------------------------------------------- | --------- |
| F-401 ※ | 建議書 / 行動投保系統串接 | 與建議書系統、行動投保系統串接，接收文稿與案件參數作為前台錄音的資料輸入來源。※ 後台另與 BPM / 行銷系統串接，共用同一整合服務框架 | Must-have |
| F-402   | 錄音編號取得         | 建議書系統傳送文稿至後端後，後端生成錄音編號並回傳建議書系統；前台取得錄音編號後方可啟動對應案件的錄音流程                 | Must-have |

## 重疊功能說明

| 功能代號  | 前台視角                     | 後台視角                            |
| ----- | ------------------------ | ------------------------------- |
| F-201 | 播放後端預先生成的 TTS 分段音檔       | 管理 TTS 音檔的生成流程、快取與清除觸發          |
| F-401 | 接收建議書系統 / 行動投保系統的文稿與案件參數 | 傳送 STT 比對結果至 BPM；回應行銷系統錄音編號清單查詢 |
